TanStack Query is available as framework-specific packages that you can install via NPM or use via CDN.

## Choose Your Framework

<Tabs>
  <Tab title="React">
    ### NPM Installation

    <CodeGroup>
    ```bash npm
    npm i @tanstack/react-query
    ```

    ```bash pnpm
    pnpm add @tanstack/react-query
    ```

    ```bash yarn
    yarn add @tanstack/react-query
    ```

    ```bash bun
    bun add @tanstack/react-query
    ```
    </CodeGroup>

    ### Compatibility

    React Query is compatible with **React v18+** and works with both ReactDOM and React Native.

    ### CDN Usage

    If you're not using a module bundler or package manager, you can use TanStack Query via an ESM-compatible CDN:

    ```html
    <script type="module">
      import React from 'https://esm.sh/react@18.2.0'
      import ReactDOM from 'https://esm.sh/react-dom@18.2.0'
      import { QueryClient } from 'https://esm.sh/@tanstack/react-query'
    </script>
    ```

    <Note>
    You can find instructions on how to use React without JSX in the [React documentation](https://react.dev/reference/react/createElement#creating-an-element-without-jsx).
    </Note>
  </Tab>

  <Tab title="Vue">
    ### NPM Installation

    <CodeGroup>
    ```bash npm

    ```

    ```bash pnpm
    pnpm add @tanstack/vue-query
    ```

    ```bash yarn
    yarn add @tanstack/vue-query
    ```

    ```bash bun
    bun add @tanstack/vue-query
    ```
    </CodeGroup>

    ### Compatibility

    Vue Query is compatible with **Vue 2.x and 3.x**.

    <Warning>
    If you are using Vue 2.6, make sure to also setup [@vue/composition-api](https://github.com/vuejs/composition-api).
    </Warning>

    ### Vue Query Initialization

    Before using Vue Query, you need to initialize it using `VueQueryPlugin`:

    ```typescript
    import { VueQueryPlugin } from '@tanstack/vue-query'

    app.use(VueQueryPlugin)
    ```

    ### Composition API Setup

    All examples in the TanStack Query documentation use [`<script setup>`](https://vuejs.org/api/sfc-script-setup.html) syntax.

    <Tip>
    Vue 2 users can use the `<script setup>` syntax using [this plugin](https://github.com/antfu/unplugin-vue2-script-setup). Check the plugin documentation for installation details.
    </Tip>

    If you prefer not to use `<script setup>`, you can translate examples to standard Composition API by moving code under a `setup()` function and returning the values used in the template.
  </Tab>

## Browser Requirements

TanStack Query is optimized for modern browsers and is compatible with:

```
Chrome >= 91
Firefox >= 90
Edge >= 91
Safari >= 15
iOS >= 15
Opera >= 77
```

<Note>
Depending on your environment, you might need to add polyfills. If you want to support older browsers, you need to transpile the library from `node_modules` yourself.
</Note>

## ESLint Plugin (Recommended)

It is highly recommended to also install the ESLint Plugin Query to help catch bugs and inconsistencies while you code:

<CodeGroup>
```bash npm
npm i -D @tanstack/eslint-plugin-query
```

```bash pnpm
pnpm add -D @tanstack/eslint-plugin-query
```

```bash yarn
yarn add -D @tanstack/eslint-plugin-query
```

```bash bun
bun add -D @tanstack/eslint-plugin-query
```
</CodeGroup>

The ESLint plugin provides rules that help you:

- Ensure exhaustive dependencies in query functions
- Maintain consistent property order in queries and mutations
- Avoid common pitfalls and anti-patterns
- Follow TanStack Query best practices

<Tip>
The ESLint plugin can catch many common issues before they become bugs in production, making it an essential tool for any TanStack Query project.
</Tip>
